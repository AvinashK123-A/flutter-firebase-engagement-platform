# Firebase Engagement Platform — Setup Guide

## Prerequisites
- Flutter SDK 3.16.0+
- Dart SDK 3.2.0+
- Android Studio / VS Code with Flutter plugin
- Xcode 15+ (for iOS builds)
- Firebase account
- Firebase CLI (npm install -g firebase-tools)

## Quick Start

### 1. Clone Repository
```bash
git clone https://github.com/AvinashK123-A/flutter-firebase-engagement-platform.git
cd flutter-firebase-engagement-platform
```

### 2. Firebase Project Setup
```bash
firebase login
flutterfire configure
# This generates lib/firebase_options.dart + native config files
```

### 3. Environment Setup
```bash
cp .env.example .env
# Edit .env with your API credentials
```

### 4. Install Dependencies
```bash
flutter pub get
dart run build_runner build --delete-conflicting-outputs
```

## Firebase Services Configuration

### FCM (Push Notifications)
- Android: google-services.json auto-generated by flutterfire
- iOS: GoogleService-Info.plist auto-generated by flutterfire
- Enable Cloud Messaging in Firebase Console
- APNs key required for iOS: Firebase Console → Project Settings → Cloud Messaging

### Remote Config
- Set up parameters in Firebase Console → Remote Config
- Fetch interval: 1 hour (production), 0 seconds (debug)
- Default values defined in lib/core/constants/remote_config_defaults.dart

### Analytics
- Automatically enabled after flutterfire configure
- Custom events defined in lib/core/constants/analytics_events.dart
- Debug mode: run with --dart-define=FLUTTER_ANALYTICS_DEBUG_MODE=true

### Crashlytics
- Automatically enabled
- Test crash: FirebaseCrashlytics.instance.crash()

### Firestore Security Rules
Deploy rules from firestore.rules:
```bash
firebase deploy --only firestore:rules
```

## Notification Topics
Available topics for subscription:
- `promotions` - Marketing & promotional content
- `updates` - App update notifications
- `alerts` - Critical system alerts
- `news` - Latest news content

## Project Structure
```
lib/
├── core/
│   ├── di/               # Dependency injection (GetIt)
│   ├── network/          # Dio HTTP client
│   ├── routes/           # GetX routing
│   ├── theme/            # Material 3 theming
│   ├── services/         # FCM, local notifications
│   └── widgets/          # Shared widgets
├── features/
│   ├── auth/             # Firebase Auth
│   ├── notifications/    # FCM notifications
│   ├── remote_config/    # Firebase Remote Config
│   ├── analytics/        # Firebase Analytics
│   ├── home/             # Dashboard
│   └── topics/           # Topic management
└── firebase_options.dart
```

## Architecture
- **Pattern**: Clean Architecture + Feature-first
- **State Management**: GetX
- **DI**: GetX service locator
- **Navigation**: GetX routing
- **HTTP**: Dio with interceptors
- **Database**: Cloud Firestore
- **Auth**: Firebase Authentication
